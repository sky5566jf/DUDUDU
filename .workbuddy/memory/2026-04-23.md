# 2026-04-23 日志

## 重启设备功能修复

**问题**：`/api/reboot` 一直不生效，原来三种方法（notify_post、posix_spawn、killall SpringBoard）都没有 root 权限，无法真正重启。

**根因**：`TVNCApiManager.mm` 的 `rebootDevice` 用的是普通 `posix_spawn`（无 persona 提权），iOS 沙盒内没有权限执行 `reboot`。

**修复方案**（参考 DevelopCubeLab/RebootTools 项目）：
- 使用已有的 `spawnRoot`（带 `posix_spawnattr_set_persona_np` root persona 提权）执行 `/sbin/reboot`
- 在 `TVNCApiManager.mm` 中加入 `#ifdef HAS_ROOT_SUPPORT` 保护的 `spawnRoot` 调用
- 同时在 imports 处加了 `#import "TSUtil.h"` 的条件编译
- 降级兜底保留：notify_post → killall SpringBoard

**改动文件**：`src/TVNCApiManager.mm`
- `rebootDevice` 方法全部重写
- imports 区新增 `#ifdef HAS_ROOT_SUPPORT #import "TSUtil.h" #endif`
