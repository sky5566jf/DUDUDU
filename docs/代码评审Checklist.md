# 代码评审 Checklist（PR 评审必过项）

> 用法：每个 PR 在评论里逐项勾选；任一项 🔴 不通过则**不得合并**。
> 配套文档：《团队技术能力提升方案》。

---

## 架构与设计
- [ ] 单一职责：一个类 / 方法只做一件事；方法超过 80 行需拆分
- [ ] 依赖清晰：私有 API 收敛到 SPI 封装层，不在业务代码裸调 `dlsym` / `objc_msgSend`
- [ ] 无重复：相同逻辑提取复用，不复制粘贴

## 健壮性（越狱 / 底层重点）🔴
- [ ] 私有 API 全部带 iOS 版本可用性检查 + 兜底路径
- [ ] 崩溃 / 信号处理只用**异步信号安全**函数（禁止在 handler 内用 Foundation / ObjC 对象）
- [ ] 网络 / 进程调用有超时、重试与失败兜底
- [ ] 指针 / 强转（`objc_msgSend`、`dlsym`）调用前判空，调用后检查返回值

## 安全 🔴
- [ ] 无敏感信息进日志（bundle id、token、密钥、URL scheme 细节按需脱敏）
- [ ] 无裸 `NSLog` 进主干，统一走 `TVNCLog`（release 关 debug）
- [ ] `dlopen` 句柄用完 `dlclose`；root 权限使用最小化

## 性能
- [ ] 无主线程阻塞（I/O、socket、launch 放后台）
- [ ] 无无谓轮询；必要时带 backoff
- [ ] 资源（timer / task / observer）有对应清理（`invalidate` / `teardown`）

## 测试与质量门禁 🔴
- [ ] 改动有对应测试，或明确说明不可测原因
- [ ] `clang-format` / `clang-tidy` 通过（CI 绿）
- [ ] 版本号 / CHANGELOG 已同步（若涉及发布）

## 规范
- [ ] 命名符合项目约定（前缀 `TVNC_` / `MatisuXCS`）
- [ ] 注释解释"为什么"而非"做什么"
- [ ] 无被抑制的编译器警告（`-Wno-*` 需评审说明）
