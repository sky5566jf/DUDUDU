## 问题 A：崩溃处理器异步信号不安全

**现状**（`app/MatisuXCS/MatisuXCS.m:7-39`）：用 `signal()` 注册，handler 内调用 `NSDateFormatter` / `NSString` / `writeToFile:`——这些**都不是异步信号安全函数**，崩溃瞬间可能二次崩溃或死锁。

**修复要点：**
1. 用 `sigaction` 替代 `signal`（`signal` 语义跨平台不一致，且不支持 `sa_mask`）。
2. handler 内**只调用异步信号安全的 `write()`**，并提前在信号安全上下文之外打开文件描述符。
3. 不触碰任何 ObjC / Foundation 对象。

```objc
// 异步信号安全的崩溃记录：仅用 write() 系统调用
#import <signal.h>
#import <unistd.h>
#import <string.h>

static int g_crashFd = -1;

static void MatisuXCSCrashHandler(int sig) {
    if (g_crashFd >= 0) {
        const char *msg = "MatisuXCS crash\n";
        write(g_crashFd, msg, (unsigned)strlen(msg));
    }
    signal(sig, SIG_DFL);
    raise(sig);
}

static void MatisuXCSSetupCrashLogger(void) {
    // 在信号安全上下文之外提前打开 fd
    NSString *path = @"/var/mobile/Media/xcs/MatisuXCS_app_crash_log.txt";
    [[NSFileManager defaultManager] createDirectoryAtPath:[path stringByDeletingLastPathComponent]
                               withIntermediateDirectories:YES attributes:nil error:nil];
    g_crashFd = open([path fileSystemRepresentation], O_WRONLY | O_CREAT | O_APPEND, 0644);

    struct sigaction sa;
    memset(&sa, 0, sizeof(sa));
    sa.sa_handler = MatisuXCSCrashHandler;
    sigemptyset(&sa.sa_mask);
    sa.sa_flags = 0;

    sigaction(SIGABRT, &sa, NULL);
    sigaction(SIGSEGV, &sa, NULL);
    sigaction(SIGBUS,  &sa, NULL);
    sigaction(SIGILL,  &sa, NULL);
    sigaction(SIGFPE,  &sa, NULL);
    signal(SIGPIPE, SIG_IGN);
}
```

> 备注：示例仅记录"发生过崩溃"。若需完整上下文（调用栈 / 线程状态），应接入 **PLCrashReporter** 等成熟库，避免自己造异步信号安全轮子——这正是资深工程"用成熟方案替代手搓"的体现。

## 问题 B：生产代码大量裸 NSLog

统一日志宏（`TVNCLog.h`），分级 + release 收口：

```objc
// TVNCLog.h
#ifdef DEBUG
#define TVNCLogDebug(frmt, ...) NSLog(@"[TVNC:D] " frmt, ##__VA_ARGS__)
#else
#define TVNCLogDebug(frmt, ...) do {} while (0)
#endif

#define TVNCLogError(frmt, ...) NSLog(@"[TVNC:E] " frmt, ##__VA_ARGS__)
#define TVNCLogWarn(frmt, ...)  NSLog(@"[TVNC:W] " frmt, ##__VA_ARGS__)
#define TVNCLogInfo(frmt, ...)  NSLog(@"[TVNC:I] " frmt, ##__VA_ARGS__)
```

**规则：**
- `error` / `warn` / `info` 进 release 包；
- `debug` 仅 DEBUG 编译期存在，release 自动消除；
- 禁止散落原始 `NSLog`，统一走 `TVNCLog`；
- 涉及 bundle id、token、密钥、URL scheme 细节的日志按需脱敏。

**替换示例**（来自 `MatisuXCS.m`）：

```objc
// 之前
NSLog(@"[MatisuXCS] Failed to load SpringBoardServices");
// 之后
TVNCLogError(@"Failed to load SpringBoardServices");
```

```objc
// 之前
NSLog(@"[MatisuXCS] LSApplicationWorkspace openApplicationWithBundleID:options:error: -> %d, error: %@", success, error);
// 之后（release 不泄露 error 细节，debug 才展开）
TVNCLogDebug(@"LSA openApp result=%d", success);
```
